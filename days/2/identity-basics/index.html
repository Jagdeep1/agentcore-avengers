Daily Challenge

Identity Basics

AgentCore Identity solves a fundamental challenge: How do you authenticate non-human identities?

Traditional authentication is designed for users clicking login buttons. Agents need something different - they need to authenticate themselves and access resources on behalf of users.

The Dual Authentication Model

AgentCore Identity handles two distinct authentication flows:

┌─────────────────────────────────────────────────────────────┐
│                    INBOUND AUTHENTICATION                    │
│           "Who is allowed to invoke this agent?"            │
│                                                              │
│  User/App → JWT Token → AgentCore Identity → Agent Runtime  │
│                              │                               │
│                    Validates: ✓ Token valid                  │
│                              ✓ User authorized               │
│                              ✓ Scopes match                  │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                   OUTBOUND AUTHENTICATION                    │
│        "How does the agent access external services?"       │
│                                                              │
│  Agent Runtime → AgentCore Identity → GitHub/Slack/etc.     │
│                         │                                    │
│              Provides: ✓ OAuth tokens                        │
│                       ✓ API keys                             │
│                       ✓ Automatic refresh                    │
└─────────────────────────────────────────────────────────────┘

Why Both?

Inbound auth protects your agent from unauthorized access. Outbound auth enables your agent to securely access resources. Both are essential for production deployments.

Core Components

1. Workload Identity

A workload identity represents your agent in the identity system:

import boto3

identity_client = boto3.client('bedrock-agentcore-identity', region_name='us-east-1')

# Create a workload identity for your agent
response = identity_client.create_workload_identity(
    name='my-agent-identity',
    description='Identity for Day 2 agent',
    tags={
        'project': 'agentcore-initiative',
        'day': '2'
    }
)

workload_identity_id = response['workloadIdentityId']
print(f"Created workload identity: {workload_identity_id}")

2. JWT Authorizer

Configure inbound authentication using JWT tokens:

# Create a JWT authorizer for inbound requests
authorizer_response = identity_client.create_jwt_authorizer(
    workloadIdentityId=workload_identity_id,
    name='my-agent-authorizer',
    issuer='https://your-identity-provider.com',
    audience='your-agent-audience',
    jwksUri='https://your-identity-provider.com/.well-known/jwks.json'
)

print(f"Created JWT authorizer: {authorizer_response['authorizerId']}")

Identity Providers

AgentCore Identity works with any OAuth 2.0 / OIDC compliant provider:

  • Amazon Cognito
  • Auth0
  • Okta
  • Azure AD
  • Google Identity

3. Credential Provider

Set up outbound authentication for external services:

# Create a credential provider for GitHub access
credential_response = identity_client.create_credential_provider(
    workloadIdentityId=workload_identity_id,
    name='github-provider',
    providerType='OAUTH2',
    oauth2Config={
        'authorizationEndpoint': 'https://github.com/login/oauth/authorize',
        'tokenEndpoint': 'https://github.com/login/oauth/access_token',
        'clientId': 'your-github-client-id',
        'clientSecret': 'your-github-client-secret',
        'scopes': ['repo', 'read:user']
    }
)

print(f"Created credential provider: {credential_response['credentialProviderId']}")

Step-by-Step Setup

Step 1: Create Your Identity Provider

For this tutorial, we’ll use Amazon Cognito as our identity provider.

# Create a Cognito User Pool
aws cognito-idp create-user-pool \
  --pool-name agentcore-day2-users \
  --auto-verified-attributes email \
  --policies '{"PasswordPolicy":{"MinimumLength":8,"RequireUppercase":true,"RequireLowercase":true,"RequireNumbers":true}}' \
  --query 'UserPool.Id' \
  --output text

Save the User Pool ID:

export USER_POOL_ID="us-east-1_XXXXXXXXX"

Create an app client:

aws cognito-idp create-user-pool-client \
  --user-pool-id $USER_POOL_ID \
  --client-name agentcore-day2-client \
  --generate-secret \
  --explicit-auth-flows ALLOW_USER_PASSWORD_AUTH ALLOW_REFRESH_TOKEN_AUTH \
  --query 'UserPoolClient.[ClientId,ClientSecret]' \
  --output text

Step 2: Configure AgentCore Identity

Create a configuration file:

# identity_config.py
import boto3
import os

identity_client = boto3.client('bedrock-agentcore-identity', region_name='us-east-1')

# Configuration
USER_POOL_ID = os.environ.get('USER_POOL_ID')
REGION = 'us-east-1'
ISSUER = f'https://cognito-idp.{REGION}.amazonaws.com/{USER_POOL_ID}'
JWKS_URI = f'{ISSUER}/.well-known/jwks.json'

def setup_identity():
    """Set up AgentCore Identity for Day 2 agent."""

    # Step 1: Create workload identity
    print("Creating workload identity...")
    workload = identity_client.create_workload_identity(
        name='day2-secure-agent',
        description='Secure agent with Identity configuration'
    )
    workload_id = workload['workloadIdentityId']
    print(f"  ✓ Workload Identity: {workload_id}")

    # Step 2: Create JWT authorizer
    print("Creating JWT authorizer...")
    authorizer = identity_client.create_jwt_authorizer(
        workloadIdentityId=workload_id,
        name='cognito-authorizer',
        issuer=ISSUER,
        audience=os.environ.get('CLIENT_ID'),
        jwksUri=JWKS_URI
    )
    print(f"  ✓ JWT Authorizer: {authorizer['authorizerId']}")

    return workload_id

if __name__ == '__main__':
    setup_identity()

Step 3: Update Your Agent

Modify your agent to use Identity:

# agent.py - Updated for Day 2
from strands import Agent
from strands.models import BedrockModel
from bedrock_agentcore.runtime import runtime_handler
from bedrock_agentcore.identity import IdentityClient

# Initialize Identity client
identity = IdentityClient()

# Initialize the model
model = BedrockModel(
    model_id="anthropic.claude-sonnet-4-20250514-v1:0",
)

# Create the agent
agent = Agent(
    model=model,
    system_prompt="""You are a secure agent with identity verification.
You can access user-specific resources based on their authenticated identity.
Always verify the user context before accessing protected resources."""
)

@runtime_handler
def handler(event, context):
    """Handle authenticated requests."""

    # Extract user identity from the request
    user_context = event.get('userContext', {})
    user_id = user_context.get('sub')  # JWT subject claim

    if not user_id:
        return {
            "error": "Unauthorized",
            "message": "Valid authentication required"
        }

    # Process the request with user context
    message = event.get("message", "")
    session_id = event.get("session_id", "default")

    # Pass user context to agent for personalized responses
    response = agent(
        f"[User: {user_id}] {message}",
    )

    return {
        "response": str(response),
        "session_id": session_id,
        "authenticated_user": user_id
    }

Step 4: Test Authentication

Create a test script:

# test_auth.py
import boto3
import requests
import os

# Get test credentials
cognito = boto3.client('cognito-idp', region_name='us-east-1')

# Authenticate a test user
def get_auth_token(username, password):
    response = cognito.initiate_auth(
        ClientId=os.environ.get('CLIENT_ID'),
        AuthFlow='USER_PASSWORD_AUTH',
        AuthParameters={
            'USERNAME': username,
            'PASSWORD': password
        }
    )
    return response['AuthenticationResult']['IdToken']

# Test the agent with authentication
def test_authenticated_request():
    token = get_auth_token('testuser@example.com', 'TestPassword123!')

    # Invoke agent with auth token
    response = requests.post(
        'https://your-agent-endpoint/invoke',
        headers={
            'Authorization': f'Bearer {token}',
            'Content-Type': 'application/json'
        },
        json={
            'message': 'What can you tell me about my account?'
        }
    )

    print(f"Response: {response.json()}")

if __name__ == '__main__':
    test_authenticated_request()

Token Vault: Secure Credential Storage

AgentCore Identity includes a Token Vault for securely storing OAuth tokens:

# Store an OAuth token
identity_client.store_token(
    workloadIdentityId=workload_id,
    credentialProviderId=credential_provider_id,
    tokenType='OAUTH2',
    accessToken='gho_xxxxxxxxxxxx',
    refreshToken='ghr_xxxxxxxxxxxx',
    expiresAt='2025-01-03T12:00:00Z'
)

# Retrieve a token (automatic refresh if expired)
token = identity_client.get_token(
    workloadIdentityId=workload_id,
    credentialProviderId=credential_provider_id
)

print(f"Access token: {token['accessToken'][:20]}...")

Automatic Refresh

Token Vault automatically refreshes expired OAuth tokens using stored refresh tokens. Your agent code doesn’t need to handle token lifecycle.

Best Practices

1. Principle of Least Privilege

Only request the scopes your agent needs:

# Good: Minimal scopes
scopes = ['repo:read', 'user:email']

# Bad: Over-permissioned
scopes = ['repo', 'admin:org', 'delete_repo']

2. Use Short-Lived Tokens

Configure token expiration appropriately:

# Recommended: Short access tokens, longer refresh tokens
oauth_config = {
    'accessTokenLifetime': 3600,      # 1 hour
    'refreshTokenLifetime': 2592000,  # 30 days
}

3. Audit Everything

Enable CloudTrail logging for identity operations:

# All identity operations are automatically logged to CloudTrail
# Access logs via:
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventSource,AttributeValue=bedrock-agentcore-identity.amazonaws.com
