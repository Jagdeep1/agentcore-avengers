Memory Strategies

Raw conversation history is useful, but extracting meaningful insights is more powerful. Memory strategies determine what information gets extracted from conversations and how it’s stored for long-term retrieval.

Built-in Strategies

AgentCore Memory provides three built-in extraction strategies:

1. USER_PREFERENCE Strategy

Extracts user preferences and settings:

# Configuration
memory_client.update_memory(
    memoryId=memory_id,
    memoryConfig={
        'longTermMemory': {
            'strategies': [{
                'type': 'USER_PREFERENCE',
                'config': {
                    'categories': [
                        'programming_languages',
                        'tools',
                        'communication_style',
                        'work_hours'
                    ],
                    'confidenceThreshold': 0.8
                }
            }]
        }
    }
)

What it extracts:

  • Language preferences: “I prefer Python”
  • Tool preferences: “I use VS Code”
  • Style preferences: “Keep explanations brief”
  • Time preferences: “I’m in PST timezone”

2. SEMANTIC Strategy

Extracts facts and entities:

# Configuration
{
    'type': 'SEMANTIC',
    'config': {
        'entityTypes': [
            'person',
            'organization',
            'project',
            'technology'
        ],
        'relationshipTypes': [
            'works_at',
            'works_on',
            'uses',
            'manages'
        ]
    }
}

What it extracts:

  • Facts: “Alex is a data engineer”
  • Relationships: “Alex works at Acme Corp”
  • Context: “Alex is working on Project Phoenix”

3. SUMMARY Strategy

Creates session summaries:

# Configuration
{
    'type': 'SUMMARY',
    'config': {
        'maxLength': 500,
        'includeTopics': True,
        'includeOutcomes': True,
        'frequencyDrivenAnnotations': True
    }
}

What it extracts:

  • Session topics: “Query optimization, PySpark”
  • Key outcomes: “Identified partitioning issue”
  • Action items: “User will test new partition strategy”

Combine Strategies

Use all three strategies together for comprehensive memory. They complement each other and serve different retrieval needs.

Memory Consolidation

When multiple memories relate to the same topic, AgentCore consolidates them:

│           │  CONSOLIDATION  │                                        │
│           │                 │                                        │
│           │  Action: UPDATE │                                        │

│                     │                                                │
│                     ▼                                                │
│  ┌──────────────────────────────────────────────┐                   │

│  │ (Updated: 2025-01-03)                         │                   │
│  │                                               │                   │
│  │ Previous: [2025-01-01] Prefers Python         │  ← Audit trail    │

└─────────────────────────────────────────────────────────────────────┘

Consolidation Actions

ActionWhen Used
ADDNew information, no existing related memory
UPDATENew info refines/extends existing memory
NO-OPDuplicate or less specific than existing
# The consolidation process is automatic, but you can configure it:
{
    'consolidation': {
        'enabled': True,
        'interval': 'END_OF_SESSION',  # or 'REAL_TIME'
        'conflictResolution': 'MOST_RECENT',  # or 'HIGHEST_CONFIDENCE'
        'auditTrail': True  # Keep history of changes
    }
}

Custom Memory Strategies

For specialized use cases, implement custom extraction:

# custom_strategy.py
from bedrock_agentcore.memory import CustomStrategy

class ProjectMemoryStrategy(CustomStrategy):
    """Extract project-specific information."""

    name = "PROJECT_CONTEXT"

    def extract(self, conversation: list[dict]) -> list[dict]:
        """Extract project-related memories from conversation."""

        memories = []

        for turn in conversation:
            content = turn.get('content', '')

            # Extract project mentions
            projects = self.extract_projects(content)
            for project in projects:
                memories.append({
                    'type': 'PROJECT',
                    'content': f"Working on project: {project['name']}",
                    'metadata': {
                        'project_name': project['name'],
                        'status': project.get('status', 'active'),
                        'extracted_from': turn.get('id')
                    }
                })

            # Extract deadlines
            deadlines = self.extract_deadlines(content)
            for deadline in deadlines:
                memories.append({
                    'type': 'DEADLINE',
                    'content': f"Deadline: {deadline['task']} by {deadline['date']}",
                    'metadata': deadline
                })

        return memories

    def extract_projects(self, content: str) -> list[dict]:
        """Extract project names from text."""
        # Use NLP or regex to find project references
        import re
        pattern = r'(?:project|working on|building)\s+([A-Z][a-zA-Z0-9\s]+)'
        matches = re.findall(pattern, content, re.IGNORECASE)
        return [{'name': m.strip()} for m in matches]

Memory Retrieval Patterns

Semantic Search

# Find memories relevant to a query
memories = memory.search_memories(
    userId='user-123',
    query='What programming languages does the user know?',
    maxResults=5,
    filters={
        'types': ['USER_PREFERENCE', 'SEMANTIC']
    }
)

for mem in memories:
    print(f"[{mem['type']}] {mem['content']} (score: {mem['relevanceScore']})")

Recency-Based Retrieval

# Get most recent memories
recent = memory.get_recent_memories(
    userId='user-123',
    limit=10,
    since='2025-01-01'
)

Type-Based Retrieval

# Get all preferences
preferences = memory.get_memories_by_type(
    userId='user-123',
    memoryType='USER_PREFERENCE'
)

# Get all facts
facts = memory.get_memories_by_type(
    userId='user-123',
    memoryType='SEMANTIC'
)

Best Practices

1. Choose Appropriate Strategies

# Customer support agent: Focus on preferences and summaries
strategies = ['USER_PREFERENCE', 'SUMMARY']

# Research assistant: Focus on facts and entities
strategies = ['SEMANTIC', 'SUMMARY']

# Personal assistant: Use all strategies
strategies = ['USER_PREFERENCE', 'SEMANTIC', 'SUMMARY']

2. Set Confidence Thresholds

# Higher threshold = fewer but more reliable memories
{
    'type': 'USER_PREFERENCE',
    'config': {
        'confidenceThreshold': 0.9  # Only very confident extractions
    }
}

3. Handle Memory Updates Gracefully

# When preferences change
def handle_preference_change(old_memory, new_content):
    """Handle conflicting preferences gracefully."""

    # Don't abruptly replace - acknowledge the change
    return {
        'content': new_content,
        'metadata': {
            'previous_preference': old_memory['content'],
            'change_reason': 'User explicitly updated preference'
        }
    }

Memory Costs

More strategies = more extraction calls = higher costs. Start with essential strategies and add more as needed.
