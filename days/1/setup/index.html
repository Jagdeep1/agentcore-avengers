Environment Setup

Before we build our first agent, we need to configure the development environment. This section ensures you have everything needed to work with Strands Agents and AgentCore Runtime.

AWS Account Configuration

Step 1: Configure AWS CLI

First, ensure your AWS CLI is configured with credentials that have appropriate permissions:

# Configure AWS CLI (if not already done)
aws configure

# Or use SSO for enterprise accounts
aws sso login --profile your-profile

Required Permissions

Your IAM user/role needs permissions for:

  • Amazon Bedrock (model invocation)
  • Amazon ECR (container registry)
  • AWS CodeBuild (for deployment)
  • IAM (to create execution roles)
  • CloudWatch Logs (for observability)

Step 2: Verify AWS Access

Test that your credentials are working:

# Verify your identity
aws sts get-caller-identity

# Expected output:
# {
#     "UserId": "AIDAEXAMPLEID",
#     "Account": "123456789012",
#     "Arn": "arn:aws:iam::123456789012:user/your-user"
# }

Enable Amazon Bedrock Models

AgentCore uses Amazon Bedrock for model inference. You need to enable model access in your AWS account.

Step 1: Access Bedrock Console

  1. Navigate to Amazon Bedrock Console
  2. Select Model access from the left sidebar
  3. Click Manage model access

Step 2: Enable Claude Models

Enable the following models:

ModelModel IDPurpose
Claude Sonnet 4anthropic.claude-sonnet-4-20250514-v1:0Primary agent model
Claude Haiku 3.5anthropic.claude-3-5-haiku-20241022-v1:0Cost-effective alternative

Model Access Approval

Model access requests are typically approved instantly, but some enterprise accounts may require additional approval time.

Step 3: Verify Model Access

# List available foundation models
aws bedrock list-foundation-models --by-provider anthropic --query 'modelSummaries[*].modelId'

# Test model invocation
aws bedrock-runtime invoke-model \
  --model-id anthropic.claude-sonnet-4-20250514-v1:0 \
  --body '{"anthropic_version":"bedrock-2023-05-31","max_tokens":100,"messages":[{"role":"user","content":"Say hello"}]}' \
  --content-type application/json \
  --accept application/json \
  /dev/stdout

Python Environment Setup

Step 1: Create Project Directory

# Create project directory
mkdir -p agentcore-day1
cd agentcore-day1

# Create virtual environment
python -m venv .venv

# Activate virtual environment
# On macOS/Linux:
source .venv/bin/activate

# On Windows:
# .venv\Scripts\activate

Step 2: Install Dependencies

Install the required packages:

# Install Strands Agents SDK
pip install strands-agents
pip install strands-agents-tools  # Includes built-in tools like calculator

# Install Amazon Bedrock AgentCore packages
pip install bedrock-agentcore
pip install bedrock-agentcore-starter-toolkit

# Verify installations
pip list | grep -E "(strands|bedrock)"

Expected output:

bedrock-agentcore              x.x.x
bedrock-agentcore-starter-toolkit  x.x.x
strands-agents                 x.x.x
strands-agents-tools           x.x.x

Step 3: Verify Strands Installation

Create a quick test file to verify Strands is working:

# test_setup.py
from strands import Agent
from strands.models import BedrockModel

# Initialize the model
model = BedrockModel(
    model_id="anthropic.claude-sonnet-4-20250514-v1:0",
    region_name="us-east-1"  # Change to your region
)

# Create a simple agent
agent = Agent(model=model)

# Test the agent
response = agent("What is 2 + 2? Reply with just the number.")
print(f"Agent response: {response}")

Run the test:

python test_setup.py

# Expected output:
# Agent response: 4

Setup Complete!

If you see a response from the agent, your environment is correctly configured. You’re ready to build your first real agent!

Project Structure

Create the following project structure for Day 1:

agentcore-day1/
├── .venv/                  # Virtual environment
├── agent.py                # Your agent code
├── tools.py                # Custom tools
├── requirements.txt        # Dependencies
└── tests/
    └── test_agent.py       # Agent tests

Create the requirements file:

# requirements.txt
strands-agents>=0.1.0
strands-agents-tools>=0.1.0
bedrock-agentcore>=0.1.0
bedrock-agentcore-starter-toolkit>=0.1.0
boto3>=1.35.0
pytest>=8.0.0

Troubleshooting

Common Issues

Issue: ModuleNotFoundError: No module named 'strands'

# Ensure virtual environment is activated
source .venv/bin/activate  # macOS/Linux
# or
.venv\Scripts\activate     # Windows

# Reinstall
pip install strands-agents

Issue: AccessDeniedException when calling Bedrock

# Check your AWS credentials
aws sts get-caller-identity

# Verify model is enabled in Bedrock console
aws bedrock list-foundation-models --by-provider anthropic

Issue: Could not connect to the endpoint URL

# Check your region setting
aws configure get region

# Ensure Bedrock is available in your region
# Supported regions: us-east-1, us-west-2, eu-west-1, ap-northeast-1
