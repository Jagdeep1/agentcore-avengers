AgentCore Runtime

Your agent works locally. Now let’s deploy it to Amazon Bedrock AgentCore Runtime - a serverless, secure hosting environment designed specifically for agentic workloads.

Why AgentCore Runtime?

FeatureLocal DevelopmentAgentCore Runtime
Session Isolation❌ Single process✅ Per-request isolation
Execution TimeLimited by local resourcesExtended timeouts for complex tasks
ScalingManual✅ Automatic serverless scaling
SecurityYour responsibility✅ Enterprise-grade sandboxing
PricingAlways running✅ Consumption-based (pay per use)

Production-Ready

AgentCore Runtime handles the infrastructure complexity so you can focus on building agent capabilities.

Step 1: Prepare Your Agent for Deployment

The AgentCore Starter Toolkit requires your agent to follow a specific structure. Let’s update our code:

# agent.py - Updated for AgentCore deployment
from strands import Agent
from strands.models import BedrockModel
from bedrock_agentcore.runtime import runtime_handler

# Import your tools
from tools import get_current_time, calculate, generate_greeting

# Initialize the model
model = BedrockModel(
    model_id="anthropic.claude-sonnet-4-20250514-v1:0",
)

# Create the agent
agent = Agent(
    model=model,
    tools=[get_current_time, calculate, generate_greeting],
    system_prompt="""You are AgentOne, a helpful AI assistant deployed on AgentCore.

You have access to the following tools:
- get_current_time: Get the current date and time
- calculate: Perform mathematical calculations
- generate_greeting: Create personalized greetings

Use these tools when appropriate to help users.
Always explain what you're doing and why."""
)

# AgentCore Runtime handler
@runtime_handler
def handler(event, context):
    """Handle incoming requests from AgentCore Runtime."""
    user_message = event.get("message", "")
    session_id = event.get("session_id", "default")

    response = agent(user_message)

    return {
        "response": str(response),
        "session_id": session_id
    }

# Local testing support
if __name__ == "__main__":
    test_event = {"message": "What time is it?", "session_id": "test-123"}
    result = handler(test_event, None)
    print(result)

Step 2: Create AgentCore Project

Use the starter toolkit to scaffold your project:

# Create a new AgentCore project
agentcore create my-first-agent --framework strands

# Navigate to the project
cd my-first-agent

This creates the following structure:

my-first-agent/
├── agent.py              # Your agent code (replace with your code)
├── requirements.txt      # Dependencies
├── Dockerfile           # Container configuration
└── agentcore.yaml       # AgentCore configuration

Step 3: Configure AgentCore

Review and update agentcore.yaml:

# agentcore.yaml
name: my-first-agent
framework: strands
runtime:
  timeout: 300          # 5 minutes max execution
  memory: 1024          # MB of memory

environment:
  AWS_DEFAULT_REGION: us-east-1

# Model permissions
bedrock:
  models:
    - anthropic.claude-sonnet-4-20250514-v1:0
    - anthropic.claude-3-5-haiku-20241022-v1:0

Step 4: Test Locally with Dev Server

Before deploying, test with the local development server:

# Start the dev server
agentcore dev

# Server starts at http://localhost:8080

Test with curl:

curl -X POST http://localhost:8080/invoke \
  -H "Content-Type: application/json" \
  -d '{"message": "What is 25 * 4 and what time is it?"}'

Fast Iteration

The dev server enables quick iteration without deploying to AWS. Use it for all your development and testing.

Step 5: Enable Observability (Optional but Recommended)

Enable CloudWatch Transaction Search before deploying:

# Enable observability in AgentCore
aws bedrock-agentcore enable-transaction-search --region us-east-1

This provides:

  • Request/response tracing
  • Tool invocation logs
  • Latency metrics
  • Error tracking

Step 6: Deploy to AWS

Deploy your agent to AgentCore Runtime:

# Deploy the agent
agentcore launch

# This will:
# 1. Build a container image
# 2. Push to Amazon ECR
# 3. Create AgentCore Runtime resources
# 4. Deploy your agent

Expected output:

Building agent container...
Pushing to ECR...
Creating AgentCore Runtime...

✅ Agent deployed successfully!

Agent ARN: arn:aws:bedrock-agentcore:us-east-1:123456789012:runtime/my-first-agent
Logs: https://console.aws.amazon.com/cloudwatch/home?region=us-east-1#logsV2:log-groups/log-group/$252Faws$252Fbedrock-agentcore$252Fmy-first-agent

Deployment Time

First deployment takes 3-5 minutes as it builds the container. Subsequent deployments are faster due to caching.

Step 7: Invoke Your Deployed Agent

Using the CLI

# Invoke with the starter toolkit
agentcore invoke --message "Hello! Calculate 15% of 85 please."

Using boto3 SDK

# invoke_agent.py
import boto3
import json

client = boto3.client('bedrock-agentcore', region_name='us-east-1')

response = client.invoke_agent_runtime(
    agentRuntimeArn='arn:aws:bedrock-agentcore:us-east-1:123456789012:runtime/my-first-agent',
    inputText='What time is it and calculate 20% of 150',
    sessionId='session-001'
)

# Parse the response
result = json.loads(response['completion'])
print(f"Agent response: {result['response']}")

Using AWS CLI

aws bedrock-agentcore invoke-agent-runtime \
  --agent-runtime-arn "arn:aws:bedrock-agentcore:us-east-1:123456789012:runtime/my-first-agent" \
  --input-text "Greet me enthusiastically as Alex" \
  --session-id "cli-test-001"

Understanding the Invocation Flow

┌───────────────┐     ┌─────────────────────┐     ┌──────────────────┐
│   Your App    │────▶│  AgentCore Runtime  │────▶│  Bedrock Model   │
│   (Client)    │     │  (Hosts Your Agent) │     │  (Claude Sonnet) │
└───────────────┘     └─────────────────────┘     └──────────────────┘
        │                      │                          │
        │  invoke-agent        │                          │
        │ ─────────────────────▶                          │
        │                      │   call LLM               │
        │                      │ ─────────────────────────▶
        │                      │                          │
        │                      │   tool_calls             │
        │                      │◀─────────────────────────│
        │                      │                          │
        │                      │  execute tools locally   │
        │                      │ ─────────┐               │
        │                      │◀─────────┘               │
        │                      │                          │
        │                      │   final response         │
        │                      │◀─────────────────────────│
        │  response            │                          │
        │◀─────────────────────│                          │

Monitoring Your Agent

CloudWatch Logs

View agent logs in the AWS Console or CLI:

# View recent logs
aws logs tail /aws/bedrock-agentcore/my-first-agent --follow

Metrics to Watch

  • Invocation count: How many times your agent is called
  • Latency: Time to process requests
  • Error rate: Failed invocations
  • Tool usage: Which tools are being called

Cleanup

When you’re done, clean up AWS resources:

# Destroy the deployed agent
agentcore destroy

# This removes:
# - AgentCore Runtime
# - ECR repository
# - CloudWatch log groups

Don't Forget to Cleanup

AgentCore charges for running agents. Always destroy unused agents to avoid unexpected charges.
