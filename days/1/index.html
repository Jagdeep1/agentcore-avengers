Day 1: The First Avenger

Welcome to The AgentCore Initiative. Today marks the beginning of your journey from agent hobbyist to production engineer.

By the end of this day, you’ll have a fully functional AI agent running in AWS AgentCore Runtime - and you’ll understand exactly why that matters.

What You’ll Build

Today we’re building AgentOne - a conversational assistant that can:

  • Respond to natural language queries
  • Use tools to perform actions
  • Run in a production-ready AWS environment

Model-Driven Development

Strands Agents uses a model-driven approach where the LLM handles reasoning and planning. You define the tools and prompts; the model decides when and how to use them.

Prerequisites Check

Before proceeding, ensure you have:

  • Code editor ready (VS Code recommended)

AWS Costs

This day will incur approximately $1-5 in AWS costs, primarily from Bedrock model invocations. We’ll cover cleanup procedures to minimize charges.

Today’s Controlled Disaster

At the end of this day, you’ll experience your first controlled failure:

CONTROLLED DISASTER

The Vanishing Context

Symptom

Your agent forgets the entire conversation after each request. Ask it your name, tell it, then ask again - it won't remember.

Root Cause

AgentCore Runtime provides session isolation, but without the Memory component, there's no persistence between invocations.

The Fix

Day 3 introduces AgentCore Memory for persistent conversation history and long-term knowledge.

Learn more in Day 3 →

This isn’t a bug - it’s a learning moment. Understanding why your agent forgets context reveals why enterprise features like Memory aren’t optional luxuries.
